Deploying a Hugo Blog with GitHub Actions and Obsidian

September 28, 2025

This guide will walk you through setting up a Hugo blog within your existing abhirudraa.github.io repository. We’ll use GitHub Actions to automatically build and deploy your site whenever you push new changes.

Prerequisites

  1. Hugo Installed: If you don’t have Hugo, install the extended version. Follow the official guide: Install Hugo.

  2. Git Installed: You need Git for version control. Install Git.

  3. GitHub Repository: You already have https://github.com/Abhirudraa/abhirudraa.github.io.

Step 1: Set Up Your Local Project

First, we’ll clone your repository and create the Hugo site structure within it.

  1. Clone your repository:

    git clone [https://github.com/Abhirudraa/abhirudraa.github.io.git](https://github.com/Abhirudraa/abhirudraa.github.io.git)

  2. Navigate into the directory:

    cd abhirudraa.github.io

  3. Initialize a Hugo site in the current directory:

    The –force flag is necessary because your directory already contains files. This command won’t delete your existing files; it will just add the required Hugo folders (content, themes, static, etc.).

    hugo new site . --force

  4. Move your existing files to the static folder:

    Hugo copies everything from the static folder directly to the final website’s root. This is how you preserve your existing index.html, intro.html, and images folder.

    # Create the static directory if it doesn't exist
mkdir -p static/images

# Move your files
git mv index.html static/
git mv intro.html static/
git mv images/* static/images/

# You can now remove the old empty images directory
rmdir images

Step 2: Add a Theme and Configure Hugo

Now, let’s add a clean theme and configure Hugo to build your site correctly.

  1. Add a theme (we’ll use ‘ananke’ as an example):

    We add the theme as a Git submodule, which is the recommended way.

    git submodule add [https://github.com/theNewDynamic/gohugo-theme-ananke.git](https://github.com/theNewDynamic/gohugo-theme-ananke.git) themes/ananke

  2. Configure Hugo:

    Create a hugo.toml file in the root of your project. Copy the content from the hugo.toml file I provided. This file tells Hugo your site’s URL, title, and which theme to use.

Step 3: Create Your First Blog Post

This is where your Obsidian workflow comes in. Hugo content lives in the content folder.

  1. Create a new post:

    Run this command to create a new Markdown file with the correct template.

    hugo new content blog/indvspak.md

    This creates the file at content/blog/indvspak.md.

  2. Edit the post in Obsidian:

    • Open your abhirudraa.github.io folder as a Vault in Obsidian.

    • Navigate to content/blog/indvspak.md and start writing!

    • IMPORTANT: In the file, you’ll see “front matter” at the top. To make the post visible, change draft: true to draft: false.

    ---
title: "Ind vs Pak"
date: 2024-09-08T12:00:00+05:30
draft: false  # <-- Change this to false to publish!
---

Your blog content starts here...

  3. Test your site locally (Optional):

    To see a live preview of your site, run the Hugo server from your project’s root directory.

    hugo server

    Then open http://localhost:1313/ in your browser. Your blog will be at http://localhost:1313/blog/.

Step 4: Set Up Automatic Deployment with GitHub Actions

This is the final step. We will add a workflow file that tells GitHub how to build and deploy your site automatically.

  1. Create the workflow directory:

    mkdir -p .github/workflows

  2. Create the workflow file:

    Create a new file named deploy.yml inside the .github/workflows/ directory. Copy the contents from the deploy.yml file I provided into this new file.

Step 5: Push to GitHub and Go Live!

Now, commit all your changes and push them to GitHub. This will trigger the deployment process.

  1. Add, commit, and push your changes:

    git add .
git commit -m "feat: Initial Hugo blog setup"
git push origin main

  2. Configure GitHub Pages Branch:

    • Go to your repository on GitHub: https://github.com/Abhirudraa/abhirudraa.github.io

    • Click on Settings > Pages.

    • Under “Build and deployment”, change the Source to GitHub Actions.

Your New Workflow

That’s it! Your setup is complete. From now on, your process for publishing a new blog post is simple:

  1. Write: Create a new .md file in the content/blog/ folder using Obsidian. (You can also use the hugo new content blog/your-title.md command).

  2. Publish: Make sure the post has draft: false in its front matter.

  3. Push: Commit and push your changes to the main branch on GitHub.

GitHub Actions will automatically take care of the rest, and your new post will be live in a minute or two!
